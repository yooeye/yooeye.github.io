Usage

The TextInput component is commonly used in forms. FormField and ReduxFormField provide a responsive layout for inputs being used in forms. Additionally, they bundle a given input with a label, a component for error messages, and a component for informational (hint) text. Use TextInput with Form and FormField (or their Redux versions) whenever possible to promote a consistent look and feel for users as well as benefit from conveniences for commonly needed form functionality.

Use the ReadOnlyInput component when displaying information in a form that can never be altered by the user. Do not use a disabled TextInput for this purpose, as it can imply that an action in the workflow might enable editing. Controls that are disabled also have lower contrast and hinder readability.

Design

TextInput may optionally display any of the ForgeIcons within the left side of the field. These should be used to reinforce the action of the field (e.g. using the ForgeIcons.Filter icon when entries in the input filter a nearby list).

API Tables

TextInput Props

Prop Type Default Description
className string Adds a class to the root element of the component
disabled bool false Adds the relevant Forge state class to the input and passes prop to native react <input />
error string Highlights the input when defined (specific string doesn't matter)
fixedWidth string Styling to be applied to width css property. Specifying this will override all other behavior that affects input width.
icon string Which icon to display inside the field
...rest Passthrough props to the <input> element

HTML API

Class Applies to Description
fe_c_text-input TextInput root Base class for the custom text input
fe_c_text-input__input input Base class for the input element in the custom text input
fe_c_text-input__icon Icon root Base class for the icon element in the custom text input
fe_c_text-input--with-icon TextInput root Required for styling specific to the input when it contains an icon
