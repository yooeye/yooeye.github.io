Textarea is for entering freeform text that is likely to be lengthier than a single line.

Usage

Textarea allows and encourages users to enter more text than a TextInput. It is most often used to collect freeform notes or comments. Use Textarea sparingly because it encourages users to enter unstructured information.

The Textarea component is commonly used in forms. FormField and ReduxFormField provide a responsive layout for inputs being used in forms. Additionally, they bundle a given input with a label, a component for error messages, and a component for informational (hint) text. Use Textarea with Form and FormField (or their Redux versions) whenever possible to promote a consistent look and feel for users as well as benefit from conveniences for commonly needed form functionality.

Design

As a user experience convenience, Textarea automatically increases in height as new lines are entered. This can be limited if needed by using the height, min-height, and max-height CSS properties.

API Tables

Textarea Props

Prop Type Default Description
className string Adds a class to the root element of the component
disabled bool false Impacts the textarea's appearance in addition to preventing editing.
error string Highlights the input when defined (specific string doesn't matter)
...rest Passthrough props to the <textarea> element

HTML API

Textarea elements (<textarea>) are styled as part of the feature class .fe_f_all.

Class Applies To Description
