  calc(state) {
    if (state.three && (state.even !== state.big)) {
      return NaN;

    const num = (state.even ? 2 : 1) * (state.three ? 3 : 1);
    return (state.big && num <= 3) ? (6 - num) : num;

Usage

Checkbox presents the user with the ability to select or deselect a single option. Multiple checkboxes can be used to display several non-exclusive options. The Checkbox is useful for enforcing structured data conventions by eliminating variability in user input.

Design

Always include a descriptive label alongside the checkbox. Descriptions are an accessibility best practice in addition to providing a larger click surface area for the component.

Consider using checkboxes when presenting fewer than six options in a field. This allows for easy visualization of the range of choices for that field.

Implementation

Checkbox creates a HTML label from the value of the description prop. This text follows the checkbox and provides context about the choice.

Use Checkbox within a FormField or ReduxFormField for features like field labels and error messaging.

This component was designed to have a consistent cross-browser appearance by using a CSS pseudo-element overlaying the HTML checkbox input element. The hidden input element is still used for all checkbox functions to maintain accessibility.

Groups of Related Checkboxes

For aesthetic or validation reasons, it may be necessary to group several checkboxes together. Here are two approaches to consider:

  1. Each Checkbox can be associated with a separate FormField. This is the simplest case because it only requires adding more FormField components with a Checkbox in the desired order.

  2. Create a custom input that houses all of the checkboxes. See the Forge Input Component repository for more detailed examples of creating custom inputs for use with FormField.

Groups with at Least One Selection

Groups of checkboxes requiring validation for at least one item to be checked can use Redux Form synchronous validation if each Checkbox is in a separate ReduxFormField.

A custom component can handle this with regular ReduxFormField validation, depending on how it combines the values of its individual checkboxes.

API Tables

Checkbox Props

Prop Type Default Description
description (required) string A descriptive label for what the Checkbox indicates; behaves like a standard input label
className string Adds a class to the root element of the component
defaultChecked bool Whether the Checkbox should be initially checked; should only be used for uncontrolled Checkboxes
disabled bool false Adds the relevant Forge state class to the input and passes the prop down to the native react <input />
error string Highlights the input when defined (specific string doesn't matter)
id string ID is sent to the root element of Checkbox so that only the description label Checkbox creates triggers the checkbox
...rest Passthrough props to the <input> element
