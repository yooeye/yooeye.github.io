Usage

Loader is a container that can be set to display a wait animation while system activity prepares content for the user. This reduces user uncertainty when the system is doing work. All content contained in Loader is blocked with an overlay while loading is in progress to prevent potential user interactions with incomplete content.

Use the Loader component when the time to load content into a page or a section of a page is likely to be longer than 1 second. Loader is indeterminate. There is no indication of progress of the load (e.g. "X percent complete"). The loading animation cycles indefinitely until dismissed programmatically.

Supply a specific label to explain to the user what activity is in progress.

Design

Loader has options depending on whether an page or section needs to be blocked during loading. Loader makes use of the ForgeIcons.AnimatedSpinner icon.

Smaller Than Viewport Section Loading

This is the best option for a page constructed from multiple sources loading content asynchronously or if only a part of the interface needs reloading. Loader fills the entire width and height of the container of the loading content and the animation is centered.

Larger Than Viewport Section Loading

Intended for sections of content taller than the viewport that allow scrolling during the load. The loading animation can be centered over the visible area of the loading content and remains in the same position relative to the viewport if the page scrolls.

Full-Page Loading

The entire page and all its content is blocked during load. For long pages scrollbars remain visible under the loader, but cannot be scrolled.

Implementation

For full-page loading, the Loader should be the first child of the React application root element in order to properly overlay all the content on the page. The assumption here is the application root has all page content as children. If this is not true (e.g. the React application is just part of the page) then the full-page loading option may not have the desired visible effect (e.g. appears as smaller than viewport section loading).

API Tables

Loader Props

Prop Type Default Description
children node All the contents that should be covered by the loader when it is "loading"
className string Adds a class to the root element of the component
fixedCenter bool false Gives the load message and animation position fixed and centers it in the currently visible area of its parent
fullPage bool false Used if the loader is the root element. Covers the entire page but leaves scollbars visible.
loading bool false Whether the loader is visible and animating; without this class the Loader has no visible style!
text string Loading message text to be placed below the loading icon; should be specific to what content is loading
zIndex number z-index override value to be sent to the blocker (overlay) element of Loader
...rest Passthrough props to the root <div> of the component

HTML API

Class Applies to Description
fe_c_loader Loader root Base class required for Loader
fe_c_loader--full-page Loader root Variation used to cover entire page content
fe_is-loading Loader root Styles the Loader to be visible and animating; without this class the Loader has no visible styling!
fe_c_loader__blocker Loader blocker element Base class required for Loader blocker element
fe_c_loader__text Loader text Styles text under the loading animation
fe_c_loader__content Loader contents Applies to contents that should be covered while content is loading; should be a child of .fe_c_loader
