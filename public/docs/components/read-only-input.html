ReadOnlyInput

ReadOnlyInput displays text that can't be edited within a FormField.

  • HTML

Usage

ReadOnlyInput shows data that cannot be altered by the user. It should be used sparingly, because the primary function of a form is to collect information, not display it.

ReadOnlyInput is intended to only be used with a FormField or ReduxFormField where its content will be associated with a label. Use the appropriate standard HTML text elements for copy introducing the form or other non-form field usages.

Design

The ReadOnlyInput content can be styled using standard HTML elements as well as other React components to provide presentation flexibility. Only use this capability for presenting static copy and images.

Implementation

Pass simple text in through the text prop.

Complex HTML/JSX content should be passed as children to the FormField component with ReadOnlyInput as the input. Technically, any content (including other inputs) can be included as children. However, those inputs won't benefit from any validation or other logic supplied by FormField. It is not recommended that interactive user interfaces be built within a ReadOnlyInput.

API Tables

ReadOnlyInput Props

Prop Type Default Description
children node Use children if more structured text is necessary
className string Adds a class to the root element of the component
id string ID for the root container; if used within a FormField, will receive the pass-through ID
text string Text to be rendered in the read-only area
...rest Passthrough props to the component's root <div> element since ReadOnlyInput is not a real field
