Usage

Icons should be used when they improve the overall readability of an application by breaking up large sections of text. They can also strengthen brand identity through consistent aesthetics.

However, a link or button with concise labeling always clearly communicates the intended action and should generally supercede the use of icons. When deciding to use an icon, consider the following:

  1. First, determine why an icon is needed and be sure it is better than text to communicate to the user. See this Nielsen Norman group article that describes the use of icons.
  2. If an icon is necessary, see if an existing Forge icon can be used. Consider providing a Tooltip to offer an explanation of the associated action if an icon is implemented without a label due to space constraints.

Use IconButton instead when an icon needs to trigger an action.

Labels with Icons

Icons should be paired with a label to ensure clarity of function. Ideally, only icons that have universal meaning for athenaNet users should be used without a reinforcing label. These icons are identified in the Index with Descriptions section.

Index with Descriptions

The following table describes the intended use cases for the included ForgeIcons.

Key for "Label Required?"

  • YES = These are the unique icons that are not commonly used or can have ambiguous meaning. Always use a label with these icons.
  • MAYBE = These icons are understood by many but may not be by some. It is preferred to put a label with the icon. When there is a limitation for having a label (i.e. space constraints), there should be a Tooltip explaining the icon.
  • NO = These icons are universally understood and do not need additional descriptions of their meaning. Labels are always helpful, but are not required.

API Tables

ForgeIcons Props

All ForgeIcons. components have the following props:

Prop Type Default Description
className string Adds a class to the root element of the component
height string Sets the height of the icon
size string Sets the size of the icon (a shortcut for setting height and width)
title string Sets the inline content of the <title> element in the svg; defaults to the icon's name
viewBox string '0 0 25 25' Sets the svg viewBox attribute; changing this is not recommended.
width string Sets the width of the icon
...rest Passthrough props to the root <svg> element

HTML API

Class Applies to Description
fe_c_icon SVG root Base class for icons
fe_c_icon__background Icon background element Base class for the background element in an icon (typically transparent)
fe_c_icon__icon Icon shape Base class for the shapes that make up the icon
fe_c_icon__icon--primary Icon shape Sets the shape to the primary style
fe_c_icon__icon--invert Icon shape Sets the shape to the inverted style
fe_c_icon__icon--secondary Icon shape Sets the shape to the secondary style when specified for that icon
fe_is-disabled SVG root Sets the icon to the disabled style
