Getting Started

Get up and running with Forge in a few simple steps.

Prerequisite Knowledge

Forge components are built in React and styled by CSS. You should be familiar with the basics of React before using them. For more advanced usage, familiarity with BEM and Sass will also be helpful.

Installation

To get started, you should already have the following installed:

  • Node.js - at least 8.9.4
  • npm - at least 5.6.0

You'll also want to make sure you have permissions to access Artifactory, our local npm package repository. An easy way to check is to visit the Artifactory web page. If you can't view the content there, you'll need to upgrade your permissions.

Using Forge-App

If you haven't created your project yet, we recommend forking forge-app, a quick-starter project based on create-react-app, but pre-configured to use Forge. The rest of the forge-app installation process is documented in the forge-app readme.

Note that if you are unable to see content at the forge-app link above you may need to request Bitbucket permissions from IdentityIQ.

From Scratch

If your project already exists or you can't use forge-app, start by creating a .npmrc file in your project's root directory and add this single line so npm knows to pull from the athenahealth Artifactory:

@athena:registry=http://artifactory.aws.athenahealth.com:80/api/npm/npm-local/

To start using Forge components in your project, add it as a dependency. In the commandline, from the root of your project:

npm install --save @athena/forge

You'll need to add React and ReactDOM (v16+) if you don't already have them. We also recommend installing validate.js and momentjs, but they aren't strictly required.

Hello World

You can now import Forge components from '@athena/forge' and render them within your own React code. If you go to App.js in forge-app, you'll see it import the heading component via:

import { Heading } from '@athena/forge';

And later render a Heading:

<Heading text="Welcome to Forge App!" />

But wait! If you aren't using forge-app, there are a couple more steps to get your page looking right.

  1. Include '@athena/forge/static/css/forge.css' in your app however you are including stylesheets (webpack, link tags, …). If you need to copy the CSS file somewhere to include it, don't forget to grab the fonts as well.
  2. Add the fe_f_all class to the root of the Forge section in your app. Forge styles are polite (scoped) and only go where invited (under fe_f_all). Hopefully you want Forge styles on the entire page, so just put it at the root element.

If you are using forge-app, the above steps are taken care of already.

Congrats! You should now be up and running with Forge.

Documentation about Heading and all the other Forge components, including live demos and copy-ready code, is available in the components section of this guide.

Advanced Topics

Forge on Part of a Page

Unlike many pattern libraries, Forge is built to be consumable only on part of a page. If that's how you're using Forge, put the fe_f_all class on the root element of the Forge section (or each section if there are multiple) instead of at the root of the page. Note that this can't be a Forge component. Forge components must be descendants of the fe_f_all element.

It's recommended to put the class fe_f_typography at the root of the page so that the entire page will at least use the same fonts.

Customizing CSS for Forge Components

With tools like Grid, the typography utilities, and the layout utilities, you can get pretty far with Forge without writing any of your own CSS. Further localized styling of Forge is discouraged as a general practice. Doing so defeats the maintainability of user interface patterns and undermines the overall goal of providing consistency to athenaNet user interfaces. Consider working with user experience and/or the Forge team to arrive at consistent, reusable solutions.

However, there may be instances (e.g. legacy integrations or prototyping) where it's necessary to make some tweaks to the default Forge visuals. This will work fine out of the box, but there are things you should know if your CSS will interact with Forge CSS:

  • If you want to use the Forge Sass constants: import '@athena/forge/sass/forge-abstracts'; You may need to add 'node_modules' to the include path for your Sass compiler, depending on your setup.
  • If you want to target a Forge component, add a custom class to the root of the component with the className prop. This prop is supported by every Forge component. Use the custom class in combination with Forge BEM classes to target elements of the component, e.g
    .my-custom-tip-class .fe_c_form-field__input {
  transform: rotate(12deg);
}
  • Most Forge CSS rules have specificity of two classes (three classes for states). Your CSS rules will need higher specificity to override them. An easy way to increase specificity is to duplicate a selector, e.g .my-heading.my-heading.my-heading to override the Forge rule .fe_f_all .fe_c_heading.
