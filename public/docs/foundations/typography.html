Typography

The Forge Design System default font is Source Sans Pro. It was selected for three reasons:

Readability

athenaNet users potentially spend all day looking at a computer screen trying to read dense blocks of information. Because of this, we’ve designed a specific font to work well with user interfaces.

Safety

For patient safety reasons we wanted to ensure each character was clearly distinguishable from others (e.g. the number one “1” is clearly distinguishable from a lowercase “l”).

Personality

While Metric (the athenahealth brand font) is an appropriate choice for promoting and selling athenahealth, within athenaNet the font should remain neutral and recede as much as possible given the seriousness and gravity of situations that can occur with medical care.

Fallback Typefaces

Arial and Sans-Serif are the fallback typefaces (in that order) if users are unable to load or choose to block the download of Source Sans Pro.

Design

All designs using Forge should be tested with the fallback fonts to account for cases where the Source Sans Pro font is unavailable (e.g. not yet served for the application or some other failure).

Link Text

A link describes and provides navigation to another page in the application. For within-page interactions, buttons are typically preferred. See the Button component for more information.

Avoid "click here" as link copy. Using the subject (or verb-subject) is more concise and properly focuses attention.

Do this: View the patient's demographic information.

Not this: Click here to view the patient's demographic information.

Links should have sentence case capitalization. The exceptions are:

  • In Streamlined patterns, standalone links are all-caps. Links that are part of, or follow, a sentence are sentence case.
  • In Classic tables, action links are lowercase.

Scoping with .fe_f_typography

Forge provides a typography feature class: fe_f_typography.

Feature classes turn on specific styles for a node and all of its descendants. This means the feature class can be applied to an ancestor of the targeted element or to the element itself. Additional feature classes include:

  • fe_f_normalize which turns on browser normalization styles
  • fe_f_all which is a shorthand for including all feature class styles Links and paragraphs are implemented as part of this typography feature class (which is also included in the "all" feature class).

HTML Styling Elements

Note that Forge does not recommend using the <b> or <i> tags since <strong> and <em> are more semantically appropriate HTML5 tags, respectively. Keep in mind that even these tags should be used sparingly. See the W3C for more information.
