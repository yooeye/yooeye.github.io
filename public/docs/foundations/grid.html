Understanding the Grid

The name grid implies a fixed matrix such as graph paper, or a table where every row has the same number of columns. While the grid could be set up and used like this, it's more useful to understand the grid as a variable number of rows each containing up to 12 columns.

In Forge, the "columns" are represented by the GridCol component and the "rows" are represented by the GridRow component. Grid as a component does not exist and is merely a term that exists to talk about layouts made with GridCol and GridRow.

Understanding Breakpoints

Breakpoints are the defined viewport widths at which GridCol may shift to provide the user with the best possible layout for consuming information. This may include changes to gutters, stacking, offsets, or order. Deciding which, if any, of these changes occurs in response to the viewport width shifting through breakpoints is the design process when using grid.

The breakpoints are defined through CSS media queries and are as follows:

Breakpoint Viewport width
small< 640px
medium640px to 1024px
large> 1024px

In general, the use of breakpoints in grid is most effective when the viewport width is the same or very close to the width of the element containing the grid layout.

Parts of the Grid

Units

  • 12 equal (by default) subdivisions of the horizontal space in a row.
  • A column (GridCol) can span up to 12 units.

Gutters

  • Space between units that provides spacing between columns
  • Interior gutters maintain a consistent width of 30px at every viewport size.
  • If necessary, the removeGuttersX property (where X is the breakpoint) can be used to remove the gutters of all columns (including edge gutters). All columns will abut with no space between them.

Edge Gutters

  • Space at the left and right edges of the grid that creates negative space on either side of a page layout
  • Ensure that content does not abut the screen edge
  • Always 15px wide regardless of breakpoint

Columns

  • Up to 12 containers that hold content or components within a GridRow.
  • The GridRow is divided into any combination of columns that together cover the full span of 12 units.
  • An individual GridCol can be 1 to 12 units wide, but the sum of GridCol units in a GridRow cannot be more than 12 units. A row with less than 12 units of columns will be represented as trailing empty space equal to the unspecified difference.
  • Successive rows of GridCol components do not need to be the same width.

Rows

  • Rows are horizontal containers of columns.
  • Each row in the grid can have varying numbers of columns.

Offsets

  • Offsets act as a column in a row but do not contain any components.
  • Offsets are used to create space between columns in a row and can be up to 11 units wide.
  • The empty column(s) appear to the left of the column with the declared offset prop.
